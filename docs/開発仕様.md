以下を **AI生成用の仕様書** として使うのがよいです。
旧PHP版は **細かい仕様の一次資料として参照**しつつ、実装方式はGoで新規設計する前提です。

# Go版 汎用メールフォーム生成仕様

## 1. 目的

旧PHP版 `f2m` をベースに、非プログラマーでも通常のHTMLフォームと簡単な設定TXTだけで設置できる、Go製の汎用メールフォームを実装する。

※旧PHP版 `f2m`ソース
docs\f2m-php.zip

旧PHP版からは、主に以下を継承する。

```text
- f2m_conf.txt の設定キー名・設定値の意味
- 複数フォーム対応
- 入力 → 確認 → 送信 → 完了 の画面遷移
- 管理者通知メール
- 自動返信メール
- CSV保存
- 添付ファイル対応の考え方
- テンプレート差し替えの思想
```

ただし、以下は継承しない。

```text
- Smarty依存
- 古いPHPMailer依存
- crypt("1234") 固定キー方式
- エラー専用画面
- HTMLを正規表現で強引に書き換える方式
- 古いPHP関数・古いセッション前提
- セキュリティ上弱い処理
```

------

## 2. 実行方式

CGI版と常駐サーバー版の両方に対応する。

```text
cmd/cgi/main.go
  共用レンタルサーバー向け。
  net/http/cgi で共通Handlerを実行する。

cmd/server/main.go
  VPS / Docker向け。
  net/http で常駐HTTPサーバーとして実行する。
```

`main.go` は薄くし、処理本体は `internal` 配下に集約する。

```text
cmd/cgi/main.go
  - 設定読込
  - 共通Handler生成
  - cgi.Serve(handler)

cmd/server/main.go
  - 設定読込
  - 共通Handler生成
  - http.ListenAndServe()
```

Ginは初期版では使わない。
理由は、CGI版と常駐版で `http.Handler` を共通利用しやすいため。

## 7. 文字コード仕様

設定TXT・HTMLテンプレート・メールテンプレートは、以下を読み込み可能にする。

```text
- UTF-8
- UTF-8 BOM付き
- Shift_JIS
- CP932
```

読み込み時の判定：

```text
1. UTF-8 BOM があれば除去
2. UTF-8として妥当ならUTF-8扱い
3. 妥当でなければCP932として変換
4. 内部処理はすべてUTF-8に統一
```

出力仕様：

```text
HTML  : 原則 UTF-8
メール: 原則 UTF-8
CSV   : F2M_CSV_CHARSET に従い UTF-8 / SJIS / CP932 を選択可能
```

```text

  ↓ 戻る

  ↓ 送信

  ↓

  ↓

  ↓
CSV保存
  ↓

```

入力チェックNG時：

```text
入力画面を再表示する
- 入力値を復元
- 画面上部にエラー一覧を表示
- 各項目付近に項目別エラーを表示
```

## 9. エラー表示仕様

エラー専用画面は作らない。
フォームHTMLを通常のHTMLとして読み込み、Go側でエラー表示を自動挿入する。

対象要素：

```text
input[name]
textarea[name]
select[name]
```

### エラー一覧

フォーム上部に全体エラー一覧を表示する。

明示プレースホルダがある場合：

```html
<!-- f2m:errors -->
```

そこに挿入する。
ない場合は、`<form>` 開始タグの直後に自動挿入する。

出力例：

```html
<div class="f2m-errors">
  <p>入力内容をご確認ください。</p>
  <ul>
    <li>お名前：入力してください。</li>
    <li>メールアドレス：メールアドレスの形式が正しくありません。</li>
  </ul>
</div>
```

### 項目別エラー

明示プレースホルダがある場合：

```html
<!-- f2m:error:mail -->
```

そこに `mail` のエラーを表示する。
ない場合は、対象入力要素の直後に自動挿入する。

例：

```html
<input type="email" name="mail">
<p class="f2m-error">メールアドレスの形式が正しくありません。</p>
```

## 10. radio / checkbox 仕様

非PG向けのため、`radio` / `checkbox` の `value` はコード値ではなく、表示文言をそのまま入れる運用を標準とする。

推奨HTML：

```html
<label><input type="radio" name="gender" value="男性">男性</label>
<label><input type="radio" name="gender" value="女性">女性</label>
<label><input type="checkbox" name="interest" value="資料請求">資料請求</label>
<label><input type="checkbox" name="interest" value="見積依頼">見積依頼</label>
<label><input type="checkbox" name="interest" value="相談したい">相談したい</label>
```

表示・保存仕様：

```text
radio:
  単一値として扱う。
  例: 性別：男性

checkbox:
  複数値として扱う。
  確認画面・メール・CSVでは「、」区切りで出力する。
  例: 興味のある内容：資料請求、見積依頼
```

エラー表示位置：

```text
radio / checkbox は同じ name が複数出るため、
項目別エラーは同名グループの最後に1回だけ表示する。
```

表示位置が崩れる場合は、明示プレースホルダを使う。

```html
<label><input type="radio" name="gender" value="男性">男性</label>
<label><input type="radio" name="gender" value="女性">女性</label>
<!-- f2m:error:gender -->
```

## 11. 入力値復元仕様

入力チェックNG時、または確認画面から戻る場合、POSTされた入力値を復元する。

対象：

```text
input type="text"
input type="email"
input type="tel"
input type="url"
input type="number"
input type="hidden"
textarea
select
radio
checkbox
```

復元ルール：

```text
text/email/tel/url/number/hidden:
  value を復元

textarea:
  本文を復元

select:
  該当 option に selected を付与

radio:
  POST値と value が一致する input に checked を付与

checkbox:
  POST値配列に含まれる value と一致する input に checked を付与
```

`password` は標準では復元しない。

## 12. バリデーション仕様

最低限、以下を実装する。

```text
必須チェック:
  F2M_CHK に含まれる項目は空禁止。

メール形式チェック:
  F2M_CHK_EMAIL に含まれる項目はメール形式を検証。

一致チェック:
  F2M_CHK_EQ の field1:field2 が一致することを検証。

添付サイズチェック:
  F2M_ATTACH_MAX を超えるファイルは禁止。

添付拡張子チェック:
  F2M_ATTACH_EXT 指定時のみ、許可一覧にない拡張子は禁止。
  未指定時は拡張子を制限しない。

添付MIMEチェック:
  F2M_ATTACH_MIME 指定時のみ、許可一覧にないMIMEは禁止。
  未指定時はMIMEを制限しない。
```

------

## 13. セキュリティ要件

```text
- 確認画面から送信する値は署名付きトークンで検証する
- 旧PHP版の固定キー方式は使わない
- メールヘッダインジェクション対策を行う
- HTML出力は必ずエスケープする
- 添付ファイル名はサーバー側で安全な一時名に変換する
- 添付ファイルの保存先は公開ディレクトリ外にする
- 一時ファイルはTTLで削除する
- honeypot項目名を自動生成できる
- 同一IPからの連投制限を行う
- 設定ファイルの読み込みエラーは行番号付きで表示する
- SMTPパスワード等の秘密情報はログに出さない
```

## 14. テンプレート仕様

Go標準の `html/template` を使う。

```text
form.html      入力画面
confirm.html   確認画面
thanks.html    完了画面
mail.txt       管理者通知メール
reply.txt      自動返信メール
```

フォームHTMLは通常のHTMLとして書ける。
Go側で以下を自動補完する。

```text
- エラー一覧
- 項目別エラー
- 入力値復元
- checked / selected 復元
```

テンプレートで使える主な値：

```text
.FormID
.Fields
.Errors
.FieldErrors
.Values
.ActionURL
```

確認画面・メールでは、全項目一覧を出力できる。

```html
{{ range .Fields }}
{{ .Label }}: {{ .Value }}
{{ end }}
```

## 15. メール送信仕様

### 管理者通知

`F2M_TO` が設定されている場合に送信する。

使用設定：

```text
F2M_FROM
F2M_TO
F2M_SUBJECT
F2M_MAIL_TMPL
```

### 自動返信

`F2M_RESV_TO_FLD` が設定されている場合に送信する。

使用設定：

```text
F2M_RESV_TO_FLD
F2M_RESV_SUBJECT
F2M_RESV_TMPL
```

### 送信方式

```text
F2M_MAIL_SENDER=smtp:
  SMTP送信

未指定:
  sendmail互換、または localhost SMTP を選択可能
```

メールヘッダでは以下を禁止する。

```text
- From / To / Subject に改行を含めない
- 宛先メールアドレスを検証する
- SMTP認証情報をログに出さない
```

## 16. CSV保存仕様

`F2M_CSV` が指定されている場合のみ保存する。

```text
初回作成時:
  1行目に F2M_JPNAME の表示名をヘッダーとして出力する。

2回目以降:
  入力値のみ追記する。
```

文字コード：

```text
F2M_CSV_CHARSET に従う。
- UTF-8
- SJIS / CP932
```

checkbox は「、」区切りで1セルに保存する。

------

## 17. 添付ファイル仕様

初期版では後回しでもよいが、仕様上は対応可能にする。

```text
F2M_ATTACH_FLD:
  添付を許可する項目名。
  未指定時は添付機能無効。

F2M_ATTACH_MAX:
  添付最大サイズ。
  未指定時は3M。

F2M_ATTACH_EXT:
  許可拡張子。
  未指定時は全拡張子を受け入れる。

F2M_ATTACH_MIME:
  許可MIME。
  未指定時は全MIMEを受け入れる。

F2M_ATTACH_TMP_TTL:
  一時保存ファイルの保持時間。
  未指定時は1h。
```

保存仕様：

```text
- アップロードファイルは公開ディレクトリ外に保存する
- 元ファイル名は表示用として保持する
- 実ファイル名はランダム名にする
- 確認画面では元ファイル名とサイズを表示する
- 署名付き送信トークンへ添付IDを含める
- 管理者通知メールにのみ添付する
- 送信後またはTTL経過後に削除する
```

## 20. 実装上の基本方針

```text
- 旧PHP版は仕様参照元として使う
- 旧PHPの実装方式はコピーしない
- Goで新規設計する
- 入口は CGI版 / 常駐版で分ける
- 処理本体は internal 配下で共通化する
- 設定ファイル形式は旧 f2m_conf.txt 互換寄りにする
- 内部文字コードはUTF-8に統一する
- フォームHTMLは非PGが編集できる通常HTMLを前提にする
- エラー表示・値復元はシステム側で自動補完する
- 崩れる箇所だけコメントプレースホルダで調整可能にする
```

## 21. AI生成時の指示文

そのまま使うなら、以下です。

```text
旧PHP版 f2m のソースを参照し、設定キー・設定値の意味・画面遷移・メール送信・CSV保存などの細かい仕様は可能な限り継承してください。

ただし、実装方式はGoで新規設計してください。旧PHPの古い実装方式はコピーしないでください。

Go版では、非プログラマーが通常HTMLと簡単な設定TXTだけで設置できる汎用メールフォームとして実装してください。

実行方式は CGI版と常駐サーバー版の両対応とし、net/http の http.Handler を共通化してください。cmd/cgi/main.go は net/http/cgi、cmd/server/main.go は net/http.ListenAndServe を使ってください。

Ginは初期版では使わないでください。

エラー専用画面は廃止し、入力フォーム画面上にエラー一覧と項目別エラーを表示してください。フォームHTMLには、Go側でエラー表示・入力値復元・checked/selected復元・honeypot項目を自動挿入してください。

radio / checkbox は、value に「男性」「資料請求」などの表示文言を入れる運用を標準としてください。checkbox は確認画面・メール・CSVでは「、」区切りで出力してください。

設定TXT・HTMLテンプレート・メールテンプレートは UTF-8 / UTF-8 BOM / Shift_JIS / CP932 を自動判定して読み込み、内部ではUTF-8に統一してください。

旧PHP版の Smarty依存、古いPHPMailer依存、crypt("1234") 固定キー方式、エラー専用画面、HTMLを正規表現で強引に書き換える方式、セキュリティ上弱い処理は継承しないでください。

署名付き送信トークン、honeypot、メールヘッダインジェクション対策、HTMLエスケープ、設定ファイルの行番号付きエラー、SMTP秘密情報のログ除外を実装してください。
```
